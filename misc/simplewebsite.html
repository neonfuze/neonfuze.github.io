SSN Database Using C++

"For legal purposes, this project does not use real Social Security Numbers and is just a simulation."

Project Explanation

This project demonstrates how to create a simulated Social Security Number (SSN) database using C++. The database is designed to generate and manage fake SSNs for testing purposes. The system includes functionalities such as adding, retrieving, and validating SSNs.

Technical Details

Main Program (`main.cpp`)

The main program processes a file containing operations to be performed on the SSN database. Each line in the file starts with an operation character ('i' for insert, 'd' for delete, 'r' for retrieve), followed by an SSN and a name. It performs these operations on a hash table and reports statistics on the number of successful insertions, deletions, and retrievals.

  • Command Line Arguments: The program expects a filename as a command line argument.
  • Operation Tracking: Keeps track of valid insertions, deletions, and retrievals.
  • Performance Measurement: Calculates and displays the elapsed time for processing the file.

Singly Linked List Implementation (`SLL.h`)

Implements a singly linked list to store nodes, where each node holds an SSN and a name. This linked list is used within the hash table to handle collisions.

  • Insertion: Adds nodes to the list if the SSN is not already present.
  • Search: Finds and returns a node with a specific SSN.
  • Removal: Deletes a node with a specific SSN.
  • Display: Outputs the SSNs of all nodes in the list.

Node Structure (`node.h`)

Defines the structure of a node used in the singly linked list. Each node contains an SSN, a name, and a pointer to the next node.

  • Data Storage: Holds SSN and name.
  • Pointer: Points to the next node in the list.

Hash Table Implementation (`hashTable.h`)

Implements a hash table using an array of singly linked lists. This table is used to store and manage SSNs and their associated names.

  • Insertion: Adds key-value pairs to the table based on the hashed value of the key.
  • Search: Finds whether a specific SSN exists in the table.
  • Deletion: Removes a key-value pair from the table.
  • Size Calculation: Returns the total number of nodes in the table.
